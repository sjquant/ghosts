# 한국어 윤문 처방집

탐지된 AI 티를 실제 문장 수정으로 옮기는 규칙이다. 탐지 없는 구간은 고치지 않는다.

## 원칙

1. 의미, 사실, 수치, 고유명사, 인용, 인과관계는 보존한다.
2. 입력 장르와 문체 격식을 유지한다.
3. 문제가 있는 구간만 국소 수정한다.
4. 문학적으로 과장하지 않는다.
5. 변경률 30% 초과는 재점검하고 50% 초과는 중단한다.

## 카테고리별 처방

### A. 번역투

| 원문 | 처방 |
|---|---|
| `X에 대해 논의한다` | `X를 논의한다` |
| `X를 통해 Y한다` | `X로 Y한다`, `X해서 Y한다` |
| `X에 있어서` | `X에서`, `X를 볼 때` |
| `X라는 점에서` | `X라서`, `X라는 이유로` |
| `X와 관련하여` | `X에`, `X를 두고` |
| `X에 기반하여` | `X로`, `X를 근거로` |
| `경쟁력을 가지고 있다` | `경쟁력이 있다`, `경쟁력이 강하다` |
| `회의를 가지다` | `회의를 하다`, `회의를 열다` |
| `판단되어진다` | `판단된다`, `판단한다` |
| `AI에 의해 생성된` | `AI가 만든` |
| `높일 수 있다` | 확정 가능하면 `높인다` |
| `합의가 이루어졌다` | `합의했다`, `합의에 이르렀다` |
| `그리고` 문두 반복 | `-고`, `-며`, 무접속으로 압축 |

### B. 영어 과다

- 괄호 영어는 첫 등장만 남긴다.
- 업계 표준 용어는 보존한다.
- `framework`, `leverage`, `robust`, `scalable`, `insight`, `impact`는 자연스러운 한국어가 있으면 옮긴다.
- 영어 인용은 원문 어감이 핵심일 때만 유지한다.

### C. 구조

- 기계적 열거는 산문으로 녹인다.
- 불릿은 정보 비교가 핵심일 때만 유지한다.
- 헤딩은 반복 구조보다 짧은 명사구나 평서형으로 쓴다.
- 문단마다 요약문으로 시작하지 않는다.
- 칼럼, 리포트, 공적 문서의 이모지는 삭제한다.

### D. 관용구

삭제 우선:

- `결론적으로`, `요약하면`, `정리하자면`
- `시사하는 바가 크다`, `주목할 만하다`
- `본질적으로`, `핵심적으로`
- `혁신적인`, `획기적인`, `압도적인`
- `~의 지평을 열다`, `~시대가 도래했다`

의미가 필요하면 원문에 이미 있는 구체 결론으로 바꾼다.

### E. 리듬

- 같은 길이의 문장이 이어지면 단문 1개 또는 긴 문장 1개를 섞는다.
- `~다` 종결이 4문장 이상 이어지면 시제와 종결을 섞는다.
- `~고 있다`는 단순 시제로 의미가 유지되면 환원한다.
- 단문만 이어지면 인접 문장을 `-며`, `-고`, `-는데`, 조건절로 묶는다.

### F. 수식과 명사화

- 정도부사는 대부분 삭제한다.
- 동의어 이중 수식은 하나만 남긴다.
- `~성`, `~적`, `~화`는 동사·형용사로 풀 수 있는지 확인한다.
- 추상어 체인은 구체 명사나 절로 바꾼다.

### G. 완곡 표현

- `~할 수 있을 것으로 보인다` -> `~로 보인다` -> `~일 것이다` -> `~이다` 순서로 낮춘다.
- 근거가 확실하면 단언한다.
- 불확실성이 실제 의미이면 완곡 표현을 남긴다.

### H. 접속사

- 문두 접속사는 대부분 삭제한다.
- `또한`은 `여기에`, `더해`, 무접속 중 하나로 바꾼다.
- `따라서/그러므로`는 인과가 자명하면 삭제한다.
- `하지만/그러나` 반복은 `그런데`, 문장 재배열, 무접속으로 줄인다.

### I. 형식명사

- `~인 것이다` -> 평서형.
- `X은 ~라는 점에 있다` -> `X는 ~다`.
- `~다는 뜻이다` -> 의미를 앞 문장에 녹인다.
- `~할 필요가 있다` -> `~해야 한다`, `~할 만하다`, 구체 행위.

### J. 장식

- 본문 볼드는 제거한다.
- 따옴표는 실제 인용과 특수 용례만 남긴다.
- 대시와 괄호는 쉼표나 문장 분리로 줄인다.

## 장르 가드

| 장르 | 허용 | 금기 |
|---|---|---|
| 칼럼·에세이 | 단문, 개인 어조 | 이모지, 과한 헤딩, 불릿 남발 |
| 리포트 | 1단계 헤딩, 통계, 인용 | hype 어휘, 이모지 |
| 블로그 | 친근한 어조, 질문형 | 기계적 열거 |
| 공적 문서 | 격식체, 문어체 | 구어체, 이모지, 불릿 남발 |

## 자체검증

1. 고유명사, 수치, 날짜, 직접 인용, 기술 약어가 바뀌지 않았다.
2. 원문 주장과 인과관계가 바뀌지 않았다.
3. 장르와 문체 격식이 유지됐다.
4. S1 패턴이 남지 않았다.
5. 변경률이 30% 이하이다.
6. 원문에 없던 비유, 예시, 주장, 인용을 추가하지 않았다.
