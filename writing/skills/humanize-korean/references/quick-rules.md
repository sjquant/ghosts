# Quick Rules

한 번의 응답 안에서 탐지, 윤문, 자체검증을 끝내기 위한 핵심 규칙이다.

## 보존 대상

다음 구간은 탐지하거나 수정하지 않는다.

- 고유명사, 제품명, 모델명, 기관명.
- 수치, 날짜, 단위, 법률 조항, 수학·화학·통계 표기.
- 직접 인용문.
- LLM, GPU, API, MCP 같은 표준 기술 약어.

## 수정 순서

1. D 관용구
2. A 번역투
3. I 형식명사
4. G 완곡 표현
5. H 접속사
6. F 수식·명사화
7. B 영어 과다
8. C/J 구조·장식
9. E 리듬

## 핵심 패턴

| ID | 심각도 | 패턴 | 처방 |
|---|---|---|---|
| A-1 | S1 | `~에 대해(서)` | 목적격 조사로 직결한다. |
| A-2 | S1 | `~를 통해/통하여` 남발 | `~로`, `~해서`, `~함으로써`로 분산한다. |
| A-3 | S1 | `~에 있어(서)` | `~에서`, `~을 볼 때`로 바꾼다. |
| A-4 | S2 | `~라는 점에서` 3회 이상 | `~서`, `~라는 이유로`로 줄인다. |
| A-5 | S2 | `~와 관련하여/관련된` | `~에`, `~의`, 목적격 조사로 줄인다. |
| A-6 | S2 | `~에 기반하여/바탕으로` 남발 | `~로`, `~을 보고`, `~를 근거로`로 바꾼다. |
| A-7 | S1 | `가지고 있다`, `회의를 가지다` | 동사·형용사로 환원한다. |
| A-8 | S1 | 이중 피동 `~되어진다` | 능동 또는 단일 피동으로 바꾼다. |
| A-9 | S2 | `~에 의해` 피동 | 행위자를 주어로 올린다. |
| A-10 | S2 | `~할 수 있다` 남발 | 확정 가능한 문장은 단언한다. |
| A-11 | S2 | `~을 위해` 남발 | `~려고`, `~위한`, 직접 절로 바꾼다. |
| A-15 | S2 | 추상 주어 + 만능 동사 | 구체 주어 또는 원인·근거 절로 바꾼다. |
| A-16 | S1 | `그/그녀/그것/그들` 반복 | 생략하거나 명사·호칭으로 바꾼다. |
| A-18 | S2 | 명사 앞 긴 좌향 수식 | 문장을 나누거나 후치 동격절로 푼다. |
| A-19 | S2 | `~에서의/~에로의/~으로의/~에의/~으로부터의` | 절로 풀어 쓴다. 단순 `~의`는 제외한다. |
| B-1 | S2 | 한글 용어 뒤 영어 괄호 반복 | 첫 등장만 병기한다. |
| B-2 | S2 | 번역 가능한 영어를 그대로 둠 | 의미를 해치지 않을 때 한국어로 옮긴다. |
| C-5 | S1 | 이모지 남발 | 칼럼, 리포트, 공적 문서에서는 삭제한다. |
| C-7 | S2 | `먼저/반면/결국` 3단 공식 | 접속사를 줄이거나 산문에 녹인다. |
| C-8 | S2 | `A인가, B인가` 반복 | 하나만 남기고 나머지는 평서문으로 바꾼다. |
| C-9 | S2 | `(1)`, `(2)`, `(3)` 인덱싱 | 산문이나 단순 줄바꿈으로 바꾼다. |
| C-10 | S1 | 콜론 부제 `X: Y` 반복 | 헤딩을 짧게 하거나 평서형으로 바꾼다. |
| C-11 | S1 | 연결어미 바로 뒤 쉼표 | 문법상 필요 없으면 제거한다. |
| D-1 | S1 | `결론적으로`, `따라서`, `이를 통해`, `그러므로`, `요약하면`, `정리하면` 반복 | 대부분 삭제하고 필요한 1-2개만 변주한다. |
| D-2 | S1 | `시사하는 바가 크다`, `주목할 만하다` | 삭제하거나 구체 결론으로 바꾼다. |
| D-3 | S1 | `본질적으로`, `핵심적으로` | 대비에 필요하지 않으면 삭제한다. |
| D-4 | S1 | hype 어휘 반복 | 삭제하거나 원문 사실로 대체한다. |
| D-5 | S1 | 의인화된 추상 주어 | 사람, 기관, 평범한 주어로 되돌린다. |
| D-6 | S1 | `~할 때다`, `~해야 한다`, `지금이야말로` 공식 결말 | 가능하면 평서형으로 닫는다. |
| D-7 | S2 | `X에서 Y로` 반복 | 하나만 남기고 일반 서술로 바꾼다. |
| E-1 | S2 | 문장 길이 균일 | 짧은 문장 또는 긴 문장으로 리듬을 섞는다. |
| E-2 | S2 | `~다` 종결 4회 이상, `~고 있다` 남발 | 종결과 시제를 섞고 단순 시제로 환원한다. |
| E-7 | S2 | 대화·구어의 말높임 불일치 | 한 화자·문단 안 말높임을 일관시킨다. |
| F-4 | S2 | `-성/-적/-화`, `-tion/-ment/-ness/-ity` 명사화 누적 | 동사·형용사로 환원한다. |
| F-5 | S2 | 추상 `~적 N` 체인 | 명사+명사 또는 구체 절로 바꾼다. |
| G-1 | S2 | `~것이다/~할 것이다` 남발 | 현재형·확정형으로 바꾼다. |
| G-2 | S2 | `~로 보인다/~인 듯하다` 남발 | 근거가 있으면 직접 말한다. |
| G-3 | S2 | `양쪽 모두`, `신중하게`, `균형` 반복 | 실제 입장이나 조건으로 바꾼다. |
| H-1 | S1 | 문두 접속사 5회 이상 | 대부분 삭제한다. |
| H-3 | S1 | `이는`, `이 점에서`, `이 관점에서`, `이 말은` 반복 | 문장 안에 녹이거나 삭제한다. |
| H-4 | S2 | `즉` 남발 | 기술 설명문이 아니면 1회 이하로 줄인다. |
| I-1 | S1 | `~인 것이다/~한 것이다` | 평서형으로 바꾼다. |
| I-2 | S2 | `X은 ~라는 점에 있다` | `X는 ~다`로 쓴다. |
| I-3 | S2 | `~다는 뜻이다/~다는 의미다` | 의미를 문장 안에 녹인다. |
| I-4 | S2 | 권고형 결말 반복 | 장르가 허용하면 평서문과 섞는다. |
| J-1 | S2 | 마크다운 볼드 남발 | 본문에서는 제거한다. |
| J-2 | 강조 따옴표 5회 이상 | 실제 인용이나 핵심 용례만 남긴다. |
| J-3 | 칼럼·리포트의 불릿 리스트 | 목록 구조가 필요 없으면 산문으로 통합한다. |

## 자체검증

1. 고유명사, 수치, 날짜, 직접 인용, 기술 약어가 바뀌지 않았다.
2. 변경률이 30% 이하이며 50%를 넘지 않는다.
3. 장르가 이탈하지 않았다.
4. 문체 격식이 이탈하지 않았다.
5. S1 잔존이 0건이다.
6. 새 비유, 수사, 주장, 인용, 예시를 추가하지 않았다.

실패한 항목이 있으면 해당 수정만 롤백하고 한 번 다시 시도한다. 그래도 실패하면 원문을 둔다.
